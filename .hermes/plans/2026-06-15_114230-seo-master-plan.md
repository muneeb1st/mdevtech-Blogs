# SEO Mastery Integration & Automated Content Generation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Transform the blogging system into an SEO-optimized engine that ranks #1 on Google for its niche, using Hermes Agent for research-driven, high-quality content that attracts organic traffic.

**Architecture:**
- **Content Generation:** Hermes-driven research → SEO‑optimized drafts → automatic publishing.
- **SEO Strategy:** Keyword‑targeted titles, semantic HTML, JSON‑LD structured data, internal linking, and XML sitemaps.
- **Data Layer:** Supabase for content versioning, SEO metadata, and analytics.
- **Deployment:** Vercel edge hosting with automatic previews.

**Tech Stack:**
- Hermes Agent (Research & Drafting)
- Supabase (Content & SEO metadata storage)
- Vercel (Hosting & CI/CD)
- Node.js (Automation & Generation)

---

### Task 1: Supabase Table Schema Extension
- **Objective:** Add SEO‑specific columns to `posts` table.
- **Files:** `automation/supabase_schema.sql` (new)
- **Steps:** 
    1. Add `meta_title`, `meta_description`, `focus_keywords`, `slug`, `canonical_url`.
    2. Add `structured_data` column for JSON‑LD output.
    3. Add `visibility_status` (public/noindex).
- **Verification:** Run `psql` query to confirm schema.

### Task 2: Hermes Research Prompt Engineering
- **Objective:** Equip Hermes with structured research queries.
- **Files:** `src/agents/hermes-research.mjs` (new)
- **Steps:** 
    1. Define a consistent prompt template for SEO keyword research, SERP analysis, and content outlines.
    2. Integrate a “SEO Score” check that must be > 0.8 before publishing.
- **Verification:** Output includes meta description, focus keywords, and a 1‑sentence SEO checklist.

### Task 3: SEO Layer in Build Process
- **Objective:** Inject SEO metadata into HTML pages and JSON‑LD blocks.
- **Files:** `src/build.mjs` (update)
- **Steps:** 
    1. Generate `meta_title` and `meta_description` from LLM output.
    2. Add `json-ld` output for `Article` schema (headline, author, datePublished, description, image).
    3. Create an `opensearch.xml` sitemap for Google indexing.
- **Verification:** Inspect built HTML source for proper `<head>` tags and JSON‑LD script.

### Task 4: Supabase Content Sync & Analytics
- **Objective:** Persist generated posts and SEO metadata to Supabase.
- **Files:** `src/lib.mjs` (extend), `src/build.mjs` (extend)
- **Steps:** 
    1. Add Supabase client initialization with env vars.
    2. After `npm run generate`, upsert the post record with SEO fields.
    3. Log analytics events (title, keywords) for future ranking analysis.
- **Verification:** Verify new row in Supabase `posts` table has SEO fields populated.

### Task 5: Vercel Deployment Configuration
- **Objective:** Prepare project for automatic preview‑deployments.
- **Files:** `vercel.json` (new)
- **Steps:** 
    1. Specify build command: `npm run generate && npm run build`.
    2. Set `outputDirectory: public`.
    3. Map environment variables: `SUPABASE_URL`, `SUPABASE_KEY`, `HERMES_API_KEY`.
- **Verification:** Run `vercel --prebuilt` locally to preview generated site.

### Task 6: Final SEO Audit & Google Indexing
- **Objective:** Validate SEO readiness and submit sitemap to Google Search Console.
- **Steps:** 
    1. Run Lighthouse audit on generated posts (target > 90 SEO score).
    2. Verify no `noindex` directives remain.
    3. Generate `sitemap.xml` and submit via Google API (curl).
- **Verification:** Lighthouse report passes, Google returns 200 OK for sitemap fetch.
